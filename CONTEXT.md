# Telegram-бот учёта расходов с AI

## Концепция

Персональный бот для ведения расходов. Юзер отправляет сумму + текст в свободной форме, AI (Claude Haiku) классифицирует трату по предопределённым категориям.

### Ключевые особенности:
- **Свободный ввод** — юзер пишет как хочет ("500 шаурма"), AI сам определяет категорию ("Еда вне дома")
- **Валюта** — по умолчанию белорусский рубль (Br), можно указать другую (100$, 50€), в отчёте сохраняются разные валюты отдельно, в диаграмме конвертируются в Br
- **КАПС-теги** — слово капсом создаёт подкатегорию для отдельного отчёта ("100 РЕМОНТ обои")
- **Детекция опечаток** — если тег похож на существующий (Левенштейн ≤ 2), бот переспрашивает inline-кнопками
- **Отчёты** — круговая диаграмма + текстовая сводка по категориям, детализация по каждой категории с датами. Основной отчёт в 3 слайдах: без тегов, по тегам, общий
- **Ручной выбор категории** — если AI не уверен или классифицирует как "Прочее", бот показывает кнопки для выбора категории
- **Пользовательские категории** — можно добавлять/удалять свои категории и подсказки
- **Удаление трат** — кнопка "Отменить" после добавления, `/delete` с пагинацией и фильтром по дате
- **Смена категории** — кнопка "Поменять категорию" после добавления траты
- **Бюджеты** — лимиты на категории в рамках периода (от зарплаты до зарплаты), прогресс при каждой трате, все валюты конвертируются в Br

### Категории (по умолчанию):
Продукты, Еда вне дома, Транспорт общественный, Такси, Жильё, Здоровье, Одежда, Подписки и сервисы, Развлечения, Красота, Подарки, Образование, Бытовые траты, Домашние животные, Накопления, Налоги, Прочее

### Команды бота:
- Текст с суммой → запись траты
- `/report` — сводка + диаграмма (дата или период: 01.05, 1 мая, 01.05-31.05)
- `/detail` — все траты по категориям с датами (без тегов)
- `/tag_report ТЕГNAME` — отчёт по тегу (можно добавить дату/период)
- `/tag_detail ТЕГNAME` — детальный отчёт по тегу
- `/category_detail Категория` — детальный отчёт по категории
- `/categories` — список всех категорий с подсказками
- `/add_category Название` — добавить свою категорию (можно с подсказками через `:`)
- `/remove_category Название` — удалить свою категорию
- `/add_hints Категория: подсказки` — добавить подсказки к категории
- `/delete` — удалить трату (можно добавить дату/период)
- `/new_period` — начать новый период бюджета (можно указать дату)
- `/set_budget Категория Сумма` — задать лимит на категорию
- `/budgets` — сводка по бюджетам
- `/help` — справка

### Форматы дат:
- `дд.мм.гггг` — полная дата
- `дд.мм` — текущий год
- `30 мая`, `30 мая 2026` — текстовый формат
- `дд.мм-дд.мм` — диапазон
- Одна дата — от неё до сегодня

## Стек

- **Язык:** JavaScript (Node.js)
- **Бот:** telegraf
- **AI:** @anthropic-ai/sdk (Claude Haiku 4.5)
- **БД:** MongoDB (mongoose), MongoDB Atlas
- **Диаграммы:** chart.js + chartjs-node-canvas
- **Курсы валют:** open.er-api.com (конвертация в BYN)
- **Деплой:** Docker, CI/CD pipeline

## Что сделано

### Этап 1 — Инициализация
- npm-проект, зависимости установлены
- Структура папок (src/bot, src/ai, src/db, src/reports, src/utils)
- Конфиг (.env, config.js с категориями)
- Модели MongoDB (User, Expense)
- Точка входа (src/index.js)

### Этап 2 — Парсер сообщений
- `src/utils/parseMessage.js` — извлекает сумму, валюту, КАПС-тег, описание
- Поддерживает форматы: "1000 продукты", "500$ РЕМОНТ обои", "$50 подарок"

### Этап 3 — Основной flow + AI-классификация
- `src/ai/classify.js` — промпт с категориями, вызов Claude Haiku
- `src/bot/handleExpense.js` — парсинг → fuzzy check → AI → сохранение → подтверждение юзеру
- Если AI не уверен или "Прочее" → кнопки для ручного выбора категории
- После ручного выбора — предложение добавить подсказку

### Этап 4 — Детекция опечаток
- `src/utils/fuzzyMatch.js` — расстояние Левенштейна, поиск похожих тегов
- `src/bot/handleCallbacks.js` — inline-кнопки для подтверждения/отклонения исправления

### Этап 5 — Отчёты
- `src/utils/parseDateRange.js` — парсинг периода (множество форматов: дд.мм, дд.мм.гггг, текстовые даты, диапазоны, одна дата = до сегодня)
- `src/reports/getExpenses.js` — запросы к БД, группировка по категориям, форматирование сумм, извлечение уникальных тегов
- `src/reports/chart.js` — генерация круговой диаграммы (png) с заголовком
- `src/bot/handleReport.js` — /report (3 слайда: без тегов, по тегам, общий), /detail (без тегов), /tag_report, /tag_detail, /category_detail

### Этап 6 — Пользовательские категории и подсказки
- `/add_category`, `/remove_category`, `/add_hints`
- `/categories` — список с подсказками (Markdown, жирные названия)
- `src/utils/categories.js` — объединение дефолтных + пользовательских категорий

### Этап 7 — Удаление и смена категории
- Кнопки "Поменять категорию" и "Отменить" после каждой траты
- `/delete` — список последних трат с пагинацией и фильтром по дате
- `recat_`, `del_`, `delmore_` callback-обработчики

### Этап 8 — Бюджеты
- `/new_period` — начало периода (сброс лимитов)
- `/set_budget Категория Сумма` — лимит на категорию
- `/budgets` — сводка с прогрессом
- `src/utils/budget.js` — расчёт прогресса с конвертацией всех валют в BYN
- Прогресс отображается при каждом добавлении траты
